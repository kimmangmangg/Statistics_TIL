# 8.1. 데이터 분석의 전체 프로세스
>## 데이터 분석의 3단계<br>
1️⃣ **설계 단계**<br>
- 무엇을 하고자 하는지 정의
- 실무자와 분석가 간의 협의<br>

2️⃣ **분석 및 모델링 단계**<br>
- 데이터 준비, 추출, 가공, 검토, 분석, 모델링
- KDD분석, CRISP-DM, SEMMA 등을 활용하여 분석 스케줄링<br>

3️⃣ **구축 및 활용 단계**<br>
- 최종 선정된 분석 모델의 성과 측정
- 필요시 IT시스템 구축<br><br>

>## CRISP-DM 분석 방법론<br>
<img width="988" alt="Image" src="https://github.com/user-attachments/assets/1dd609a4-4f69-4350-950b-31fd2bbebda8" /><br>
- **비즈니스 이해** : 현재 비즈니스 상황 평가, 데이터 마이닝 및 프로젝트 목표 결정
- **데이터 이해** : 데이터 품질 확인 및 탐색을 통해 데이터 이해
- **데이터 준비** : 데이터 선택, 정제 및 통합
- **모델링** : 모델링 기법 선정 및 테스트 디자인 생성, 모델 생성 및 평가
- **평가** : 결과 평가, 프로세스 검토, 다음 단계 결정
- **배포** : 모니터링 및 유지 관리 계획, 배포 계획<br><br>

>## SAS SEMMA 분석 방법론<br>
<br><img width="974" alt="Image" src="https://github.com/user-attachments/assets/168377a9-e8c8-4651-84db-03f826fcb35c" /><br><br>
- **Sampling** : 분석용 데이터 추출(대표성을 가지는 표본 추출 등)
- **Exploration** : 통계치 확인, 변수 간 관계성 파악, 변수 선정, 데이터에  맞춰 비즈니스 아이디어 도출 및 분석 방향성 수정
- **Modification** : 결측치 처리, 로그변환, 구간화, 주성분 분석, 최종 변수 선정, 새로운 변수 생성
- **Modeling** : 비즈니스 목적에  맞는 모델 선정
- **Assessment** : 모델 예측력 등 성능 비교, 비즈니스 인사이트 적용, 추가 데이터 분석 진행<br><br>

### Summary <br>
**[초반부]** : 비즈니스 문제 인식, 해결 방향 정의, 데이터 탐색<br>
**[중반부]** : 데이터를 목적에 맞도록 수집 및 가공, 머신러닝 모델 사용<br>
**[후반부]** : 분석 결과 검토 및 검증하여 실제 환경에 적용<br><br>
➡️ 이후에도 적용한 방법에 대해 효과 모니터링, 성과 측정, 보완 단계 지속해 나가야 함<br><br>
